# BUILDERBRIEF — Vertux Nexus Product Site

## Роль сайта

Публичная презентация Vertux Nexus и инструмент для демонстрации продукта на Zoom-созвоне. Сайт должен создавать желание получить Nexus, а не повторять основной сайт студии и не превращаться в форму обсуждения проекта.

## Продукт

Vertux Nexus — система управления персональными бизнес-продуктами компании:

- Workspace под реальный процесс.
- Call Intelligence: транскрипция, AI-суфлёр и тренер по звонкам.
- AI-виджет: диалоги, база знаний, качество ответов и лиды.
- Local Node для 1С, файлов, резервных копий и закрытых операций на ПК компании.
- Managed Cloud и Hybrid-размещение.
- Клиентский кабинет: продукты, версии, подписка, AI usage и поддержка.

## Выбранное направление

`Cinematic Product Portal` — футуристичный продуктовый сайт с фирменным графитово-фиолетовым визуалом Vertux.

- Чёрный и графитовый фон чередуются по секциям.
- Фиолетовый свет — фирменный акцент, а не сплошная заливка.
- Крупная типографика и крупный читаемый интерфейс продукта.
- Оригинальный знак Vertux из бренд-ассетов.
- Никаких эмодзи и декоративного микротекста, который невозможно прочитать.
- Никаких названий клиентских компаний на публичной странице.
- Один корректный watermark в footer: «Сделано в студии Vertux».

## Референсы и их роль

- Google Antigravity — ощущение продукта из будущего и кинематографичная подача.
- FunPay Pulse — уверенная продуктовая структура: интерфейс, загрузка, обновления, кабинет.
- Flowty — последовательное раскрытие предмета и слои, двигающиеся вместе со страницей.
- Fluently — темп прокрутки, большие продуктовые сцены, постепенное появление элементов.
- Clyde — карточки и интерфейсные слои, которые собираются в цельную систему.

Референсы не копируются по палитре, графике или композиции. Из них берётся режиссура движения и продуктовая уверенность.

## Сценарий презентации

1. Hero: короткий оффер и большой экран Nexus.
2. Во время прокрутки экран поднимается из перспективы, вокруг последовательно появляются продукты.
3. Внутри экрана можно вручную переключить Overview, Calls + AI, AI Widget и Local Node.
4. Контраст «таблицы / чаты / звонки / документы» превращается в единый Nexus.
5. Три продуктовых мира показывают Workspace, AI Intelligence и Operations.
6. Размещение объясняет Local, Hybrid и Managed Cloud без технической перегрузки.
7. Клиентский кабинет показывает подписку, продукты, версии и usage.
8. Финальный CTA ведёт к обсуждению и Windows-версии.

## Структура

- `index.html` — главная продуктовая презентация.
- `updates.html` — честная история изменений и статусы направлений.
- `download.html` — Windows и Web Access без фальшивой ссылки на несуществующий установщик.
- `account.html` — интерфейс входа по приглашению; до подключения backend форма не отправляет данные.
- `assets/site.css` — единая дизайн-система и адаптив.
- `assets/site.js` — RU/EN, продуктовые режимы, scroll-driven движение и доступность.

## Языки

- Полный RU/EN-переключатель.
- Выбор сохраняется в `localStorage`.
- Заголовок страницы меняется вместе с языком.
- Английская версия не должна оставлять русские продуктовые абзацы.

## Motion

- Большой экран Nexus поднимается из перспективы по позиции прокрутки.
- Четыре продуктовые карточки появляются последовательно.
- Карточки и секции получают controlled reveal.
- Два интерактивных переключателя: режим интерфейса и продуктовый мир.
- Бесшовная строка дублируется двумя одинаковыми группами.
- `prefers-reduced-motion` отключает непрерывную анимацию.
- Никаких тяжёлых видео и бесконечных WebGL-сцен на текущем этапе.

## Честность продукта

- Не писать «скачать», если файла установщика ещё нет.
- Текущий CTA: «Запросить установку».
- Вход по приглашению не изображает работающую авторизацию без backend.
- Демонстрационные цифры находятся только внутри явно демонстрационного интерфейса.
- Planned/Design Locked и Released не смешиваются в changelog.

## Responsive

- Desktop — полная презентационная сцена.
- Tablet — горизонтальная навигация интерфейса и упрощённые спутники.
- Mobile — без боковых всплывающих карточек и без тяжёлого 3D; ключевые режимы остаются кликабельными.
- Минимальный размер основного текста — 16 px; UI-текст в демонстрационном окне не должен превращаться в декоративные точки.

## Критерии готовности

- Нет упоминаний клиентских проектов.
- RU/EN покрывает все элементы с `data-i18n`.
- Marquee не имеет пустого хвоста и не прыгает на цикле.
- Все локальные ссылки открываются.
- JavaScript проходит syntax check.
- Интерфейс удобен для последовательного рассказа на демонстрации.
- После визуального QA исправлены desktop и mobile состояния.
